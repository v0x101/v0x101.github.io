## 19-Mar-2026

I wanted to upgrade my Raspberry Pi 4b to latest OS. So downloaded Raspberry Pi OS Full (64-bit) from [here](https://www.raspberrypi.com/software/operating-systems/).
- The image was compressed using xz. Decompressed it using command `xz -dkcvv raspios-trixie-arm64-full.img.xz > raspios-trixie-arm64-full.img`. The command line flags indicate:
  - `-d, --decompress    ` force decompression
  - `-k, --keep          ` keep (don't delete) input files
  - `-v, --verbose       ` be verbose; specify twice for even more verbose
  - `-c, --stdout        ` write to standard output and don't delete input files
- The image was copied to SD Card using following procedure:
  - Connect the SD Card to computer using adaptor
  - Unmount all volumes from SD Card - SD Card should be visible in the output of `lsblk`, but should not be mounted as per output of `df -kh`
  - Execute command: `sudo cp -v ./raspios-trixie-arm64-full.img /dev/sdd` (must use raw device name)

After instlling the OS, I was running `apt upgrade` on Raspberry Pi OS 13.4, and my console terminal closed all of sudden, but `apt upgrade` was still running. Further, it so happened that `apt upgrade` was asking for my input when this happened (as evidenced by `/var/log/apt/term.log`) - pretty bad timing. Here is how I recovered.
- View real-time progress via logs: `sudo tail -f /var/log/apt/term.log`
- Google Gemini suggested: _If you see a process named whiptail, it means the upgrade is waiting for you to answer a prompt (like "Keep or replace configuration file?")_ However, I missed to check this at the time.
- Stopped the stuck processes: Find PIDs using `ps aux | grep -E "apt|dpkg"` and kill the processes using `sudo kill -9 <PID>`
- Clear the Locks - when killed abruptly, `apt` leaves lock files that prevent you from starting a new instance
  - `sudo rm /var/lib/dpkg/lock-frontend`
  - `sudo rm /var/lib/apt/lists/lock`
  - `sudo rm /var/lib/dpkg/lock`
- Fix the interrupted installation and finish the upgrade
  - Configure partially installed packages: `sudo dpkg --configure -a`
  - Fix any broken dependencies: `sudo apt install -f`
  - Finish the upgrade: `sudo apt upgrade`

To avoid such mishaps, Gemini suggested to use a terminal multiplexer. So I chose __tmux__.
